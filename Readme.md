## About

**CGOW** is a work-in-progress open-world demo built by [Sanjay Meher](https://cggodotassets.shop) as a technical showcase — every system in this game is a production-tested asset available at the store. The game serves as a living portfolio: if you see something in the game that you want in your own project, it's available as a drop-in Godot 4 system.

**Story:** Players arrive in a corrupt country after being tricked by a fake job agency. With no documents, no money, and corrupt police demanding bribes, they must work for Rico — a smuggler with his own code — and earn trust to survive and escape. Missions are desperate, morally complex, and dynamically generated by AI.

## Core Systems

### 🧠 AI Mission System (`MissionGenerator.gd` + `MissionManager.gd`)
The heart of the game. Missions are generated dynamically using **Groq (Llama 3.3 70B)** or **Gemini** and saved as JSON so they persist across sessions.

- `assign_or_generate` checks local JSON files before hitting the API — no wasted API calls
- Per-NPC tone and story context baked into every prompt (Rico = smuggler, David = lawyer, Marco = robber, etc.)
- 13 supported task types: `reach_marker`, `knock_target`, `escort_npc`, `deliver_vehicle`, `survive_duration`, `talk_to_npc`, `lose_wanted_level`, and more
- Auto-retry with exponential backoff on API failure
- Full JSON validation + task type sanitization (e.g. `kill_target` → `knock_target`)
- Recent mission cache to avoid repetition
- Mission variation generator for replay value
- Co-op role assignments (`muscle`, `infiltrator`, `analyst`) per task
- Supabase backend for cross-session mission progress, attempt counts, and job cooldowns

### 🤖 NPC Conversation System (`GameNPC.gd`)
A full conversational NPC framework — not just a character script.

- 5 built-in NPC presets (Rico, David, Marco, Charlie, Henry) with distinct personalities, trust tiers, and welcome lines
- Trust-based dialogue: first time → doing well → trust building → respected → uncooperative
- Dual AI provider support: Groq and Gemini, switchable per NPC via Inspector
- Voice support: NPCVoice / ElevenLabs integration
- Full Solo and Multiplayer support with RPC calls built in
- Moving and Static NPC behavior modes
- Obstacle avoidance, flee behavior, and death handling
- Mission talk tasks — NPC can trigger specific mission stages via conversation

### 🚗 Multiplayer Vehicle System (`car.gd`)
- Driver + passenger seat with peer ID tracking
- Engine and skid audio with dynamic pitch scaling by speed
- Networked via **CGRelay** WebSocket relay — position, rotation, and state synced
- Entry/exit area detection with player proximity logic
- Smooth interpolation for remote player car positions
- Handbrake, steering, reverse, and friction all fully configurable via Inspector

### 🎒 Inventory & Weapon System (`PlayerInventory.gd` + `WeaponController.gd`)
- Weapon modes: Gun, Melee, Grenade, None
- Primary gun slot with mag cache — swapping weapons saves current ammo
- Headshot multiplier (2x damage), raycast-based hit detection
- Reload system with timer-driven animation sync
- Ammo reserve tracking, no-ammo signal for UI feedback
- Signals for every state change: `gun_equipped`, `reload_started`, `ammo_changed`, `hit_registered`
- Remote player weapon state handled separately — only local player processes fire/reload

### 🧍 Player Controller (`player.gd`)
- Walk / Run / Idle / Fire / Death / Reload animation state machine
- Full mobile support: touch joystick, touch camera, fire and run buttons
- Blood effect system with configurable scatter radius and fade
- UI focus stack for managing inventory, map, and HUD layering
- Respawn system with black screen transition
- Med and drink item buttons with inventory integration
- Multiplayer-aware: `is_local` flag gates all input and processing for remote peers

### 👾 Enemy AI (`enemy.gd`)
- NavigationAgent3D pathfinding with stuck detection and escape teleport
- Detection range + attack range with raycast line-of-sight check
- Visibility check batched every 6 frames — no per-frame raycast overhead
- Health bar as 3D MeshInstance with dynamic material color
- Walk, attack, and death animation states
- Configurable fire rate, damage, move speed, and rotation speed via Inspector
- Wave-based spawning support via `WaveConfig` resource

### 🗺️ Minimap & Full Map (`cam.gd`)
- Dual camera setup: rotating minimap (nav_cam) + draggable orthographic full map (full_cam)
- Minimap rotates with player direction — GTA-style
- Full map supports pinch zoom and touch drag with clamped bounds
- Mission waypoint icon auto-updates from `MissionManager.get_active_task_waypoint()`
- Custom waypoint placement: tap on full map to drop a marker
- Reacts to all MissionManager signals: started, completed, failed, abandoned

### 🍕 Job NPC System (`jobNPC.gd` + `customer.gd`)
- Gary (Pizza Delivery) and Lewis (Taxi) job NPCs with AI-generated job dialogue
- `customer.gd` spawns at random offset from marker with ground raycast snap
- Detects specific vehicle type proximity (pizzabike / taxi)
- Freeze-and-complete interaction logic on vehicle arrival
- Lewis as taxi passenger: attaches to vehicle seat offset during ride

### 👤 Player Profile (`PlayerProfile.gd`)
- Supabase-backed player identity: level, money, houses, cars, stats
- Debounced save system — multiple calls within 2 seconds collapse into one write
- Retention system integration: each manager (currency, achievements, etc.) owns its own profile slice
- Avatar support: bundled avatars + custom user avatars from `user://avatars/`
- Last-user persistence for auto-login on relaunch
- LevelSystem integration via `register_level_system()`

## Setup for Collaborators

### Requirements
- [Godot 4.x](https://godotengine.org/download)
- Android export templates (for Android builds)

### Getting Started

```bash
git clone https://github.com/sanjaymeherdev/Godot-Open-World.git
# Open in Godot 4 → File → Open Project → select folder
```

### API Keys Required

> ⚠️ Never hardcode API keys. Use empty strings in code and set them via the Inspector or a local config file that is gitignored.

| Key | File | Purpose |
|---|---|---|
| Groq API Key | `MissionGenerator.gd`, `GameNPC.gd`, `jobNPC.gd` | AI mission + NPC dialogue |
| Supabase Anon Key | `MissionManager.gd`, `PlayerProfile.gd` | Player progress + profile storage |

Get a free Groq key at [console.groq.com](https://console.groq.com)
Get a Supabase project at [supabase.com](https://supabase.com)
